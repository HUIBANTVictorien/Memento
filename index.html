MEMENTO

Ligne de commande


Pour créer un dossier :

mkdir "nom du dossier"

mkdir "nom du dossier" "nom du dossier" "nom du dossier" (créer plusieurs dossiers dans la même racine)

mkdir -p (parents)"nom du dossier"/"nom du dossier"/"nom du dossier" etc ...


Pour créer un fichier :

touch "nom du fichier.extention"


Pour lancer une application :

"nom de l'application" lance l'application nommée (firefox/atom/slack etc ...)

atom index.html (lance index.html dans atom (possibilité d'ouvrir plusieurs fichiers en même temps)il faut se trouver dans le dossier)

firefox "nom du fichier html" ouvre le fichier dans un nouvel onglet


Pour afficher une liste des dossiers, des fichiers et les manuels :

ls (affiche les dossiers et fichiers du dossier courant)

ls -a (affiche les dossiers/fichiers cachés)

ls -l (affiche les dossiers/fichiers en liste longue)

ls -al (affiche les dossiers/fichiers(même cachés) en liste longue)

man "nom du manuel recherché"


Pour afficher le chemin du dossier :

pwd


Pour naviguer dans les dossiers :

cd "nom du dossier"

cd~ permet de revenir au dossier user(victorien)

cd .. permet de revenir au dossier précédent


Pour passer en mode super user :

sudo -s


Pour gérer les paquets :

apt update (met à jour les paquets)

apt upgrade (met à jour l'OS)


Poour rerchercher des fichiers :

apt search "nom recherché"

apt search "nom recherché" | (pipe) grep "mot recherché"

GIT


Initialisation dans répertoire :

git init


Ajouter fichier pour indexation :

git add


Cloner un dépôt distant :

git clone git:"URL" (espace "***"pour renommer)


Vérification d'état du dosser git :

git status


Voir ce qui a été modifié après git status :

git diff


Ignorer des fichiers :

cat .gitignore .(type de fichier) ou ~ pour les fichiers temporaire


Validation des modifications pour préparer envoi sur dépôt (après un git add) :

git commit -m "***"


Effacer fichier git :

git rm (/!\ -f pour forcer/!\)


Visualiser les validations :

git log ( -p pour voir les différences entre les validations)


Mouvoir les fichiers :

git mv


Statistiques des commits :

git log --stat


Format différent de git log (voir PrettyFormat) :

git log --pretty format=(voir lien au dessus)


Annulation du dernier commit :

git commit --amend(/!\ supprime définitivement le commit/!\)


Ajouter un dépôt distant :

git remote add"URL"


Pousser du travail sur un serveur distant :

git push origin master(master étant le nom de la branche à push, peut changer selon les créations de branches)(en ayant enregistrer le remote)


Inspecter un dépôt distant

git remote show "nomdistant"


Renommer branche/dépôt :

git remote rename %target ("***")


Lister étiquettes(tag) :

git tag

git tag -l "v1.4.*"(liste les tags v1.4.x)


Créer des tags annotés :

git tag -a v1.4 -m 'Nom' ( -m spécifie le message du tag)


Créer des tags signés :

git tag -s


Vérifier un tag signé :

git tag -v "nom tag"


Ajouter une branche :

git branch "***"


Changer de branche :

git checkout "nom branche souhaitée"


Fusionner branches :

git merge

Vhost

/!\Dans le fichier.conf ne pas mettre Options +Indexes au dessus de allow/!\

/!\Celà permet de voir les fichiers non cachés ("index of ***")/!\


Création de Vhost :

Se mettre en root (sudo -s)

Aller dans /etc/apache2/sites-available/

Nano "nom".conf copier coller :


<VirtualHost *:80>

ServerAdmin webmaster@monProjet

ServerName www.monProjet

ServerAlias monProjet

DocumentRoot /home/nom_utilisateur/www/monProjet

<Directory /home/nom_utilisateur/www/monProjet/>

AllowOverride All

Require all granted

</Directory>

ErrorLog /var/log/apache2/monProjet-error.log

LogLevel warn

CustomLog /var/log/apache2/monProjet-access.log combined

ServerSignature Off

</VirtualHost>


Bien changer les noms de fichiers et le chemin du dossier

Activez le .conf avec a2ensite "***"

Relancez apache2 avec systemctl restart apache2

Aller dans etc/, nano hosts et ajouter 127.0.0.1 "***"
